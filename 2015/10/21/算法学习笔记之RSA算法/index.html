算法学习笔记之RSA算法

发表于 2015-10-21   |   分类于 算法   |   阅读次数

1. 起因

　　因为我们每周要上算法课，在老师留的其中一道题目当中涉及到了RSA算法，在网上详细学习了一下RSA算法，同时也搞懂了其中的证明（强烈推荐这篇文章）。其实我的疑问是：在证明之前是如何判断出来一定可以证明出来，难道是不断的尝试？对此我感到很神奇，同时也十分膜拜三位创造该算法大神的超高智商和精妙的证明。

2. 题目介绍

题目链接:

　　http://acm.hust.edu.cn/vjudge/contest/view.action?cid=96025#problem/D

题目大意:

　　题目简单来说就是RSA密码解密，题目中给出了公钥(n,e)以及加密之后的数字c，那么让我们求出加密前的数字m。

3. 思路分析

　　这题目是显然的RSA解密，因为给出的数据范围很小，那么可以暴力求出我们所需要的解。首先我们根据RSA加密知道，我们要解密的话就必须知道私钥(n,d)，这里的关键就是求出d。那么可以画出逻辑递推图: 　　　　　

𝑒和𝜑(𝑛)互素 => 𝑒𝑑 ≡1 (𝑚𝑜𝑑 𝜑(𝑛)) => 𝑒𝑥+𝜑(𝑛)𝑦=1

然后利用扩展欧几里得定理计算出x，那么很容易知道d = x，我们根据𝐶𝑑 = 𝑚 (𝑚𝑜𝑑 𝑛)计算出m，也就是𝑚 = 𝐶𝑑 𝑚𝑜𝑑 𝑛,这一步需要用到快速幂算法来计算(有一些表达式具体证明见RSA算法)。

4. 算法步骤

  • 步骤1：首先对32000以内的数进行打表，找出所有的素数
  • 步骤2：遍历步骤1求出的素数，找到两个素数p和q，使其符合n = pq
  • 步骤3：对e和𝜑(𝑛)使用扩展欧几里得定理算出d
  • 步骤4：利用快速幂算法求𝑚 = 𝐶 𝑑 𝑚𝑜𝑑 𝑛

5. 算法复杂度

　　步骤一的时间复杂度为O(nlogn)，此时n = 32000，打表时间可以
忽略；步骤二这个计算不好说时间复杂度，但是因为给出的n很小，而且找到的素数个数不是很多，那么这一步的时间也是可以忽略的；步骤三扩展欧几里得定理的时间复杂度为O(loge)，因为𝑒 < 𝜑(𝑛)，这是有证明的，就不详细解释了；步骤四快速幂的时间复杂度为O(logd),d是我们私钥里面的乘法逆元。
　　所以综上,在每次循环计算RSA解密的时间复杂度为O(logn)，这里的n是e和d的最大值，再加上外面的循环k次，那么总的时间复杂度为O(klogn)。
