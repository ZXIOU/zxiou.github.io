C++学习笔记之class与struct区别

发表于 2016-01-23   |   分类于 C++   |   阅读次数

1. 前言

　　因为我们CS的程序设计老师是学校ACM校队的教练，所以我们的程序设计课基本是算法和编程，很少涉及语言的深入介绍，C与C++基本都是自学的，自己看书自己琢磨，遇到不会的问题会请教一些保送生大神^_^，现在想一想大一那段时间也是相当辛苦的，不过也练就了自身超强的学习能力。C++是一门博大精深的语言，绝对不是能够熟练使用语言的基本语法就能算得上是精通，里面内部的实现需要我们不断的探索与经验的积累。言归正传，以前使用C++写struct和class时，并没有详细的想过这两者的区别，只是在使用封装的时候要用class，当要自定义数据集合的时候使用struct结构体（其实C++ Primer里面也叫这个是类），前几天看到了Lippman（C++ Peimer作者之一的大神级人物OTZ）的一句调侃话：

“嘿，你知道吗，struct那个关键词，其实没什么用……”

这句话看起来有些搞笑和大神的嘲讽，不过肯定有其中的道理，我决定好好探究一下class和struct的区别和联系。

2. C中struct与C++中class的区别

　　C是一种过程化的语言，struct只是作为一种复杂数据类型定义，struct中只能定义成员变量，而不能定义函数，也就不可能继承也没有权限的概念，那么更不可能不能面向对象编程，仅仅只是一种自定义的复合数据结构而已。

3. C++中struct与C++中class的区别

　　在C++中，struct可以继承、设置权限限制、定义函数，甚至也有虚函数，看起来class有的struct也有嘛，但是他们有下面这几点区别：

  • 默认访问权限不同。class的默认访问权限为private权限，而struct默认是public权限
  • 默认继承权限不同。class的默认继承权限为private权限，而struct默认是public权限
  • class能够作为模板参数，而struct并不能

从上面可以看出在C++中，class的功能更加全面，且能够保证封装性和安全性，struct的默认public并不利于面向对象的封装。struct能够在C++中继续保留，语言设计者的初衷主要是为了兼容C，让想要转C++的C程序猿能够更加容易的去适应和理解C++，而并不打算让C++开发者去真正使用它，这也是上面Lippman的调侃所在。所以如果在C++下开发，我个人建议尽量使用class而不是struct，因为class的功能更加强大也符合C++语言面向对象的封装性和安全性。

4. 一些想法

　　涉猎广泛，多阅读多思考确实是一件好事，如果我没有看到那一句调侃，可能我并不会去详细研究class和struct的区别，也有可能在别的学校很多老师都会讲，但是我们的老师并没有讲，所以我通过广泛的阅读弥补了这一差距。这也让我认识到，在学习的道路上不能鼠目寸光，要不断的开阔眼界，阅读任何只要自己感兴趣的只是总会有收获的。
